# User Flows

작성일: 2026-07-05

## 1. 기본 제품 루프

핵심 루프:

수집 -> 프로젝트 연결 -> 태스크 분해 -> 현재 환경에 맞게 변환 -> 추천 -> 실행 -> 로그 -> 프로젝트 이해 업데이트

이 루프가 제품의 중심이다.

## 2. 매일 앱을 여는 흐름

### 목적

사용자가 앱을 열었을 때 바로 "지금 무엇을 해야 하는지"를 볼 수 있게 한다.

### 흐름

1. 사용자가 앱을 연다.
2. 오늘 작업 화면이 열린다.
3. 상단 실행 컨텍스트 바가 보인다.
4. 사용자는 현재 환경이 맞는지 확인한다.
5. AI 추천 작업 3개가 보인다.
6. 각 작업에는 추천 이유가 한 줄로 표시된다.
7. 사용자는 하나를 시작한다.

### 화면에 보여야 하는 것

- 실행 컨텍스트 바
- 추천 작업 3개
- 빠른 수집 입력
- 이번 주 전진 프로젝트 요약

## 3. 해야 할 일이 떠오르는 흐름

### 목적

떠오른 할 일을 놓치지 않고 수집하되, 사용자가 분류 부담을 지지 않게 한다.

### 흐름

1. 사용자가 빠른 수집 입력창에 자유롭게 입력한다.
2. AI가 입력 유형을 추정한다.
3. AI가 프로젝트 연결을 시도한다.
4. 확신이 낮으면 질문 하나를 한다.
5. 사용자가 짧게 답한다.
6. AI가 프로젝트 맥락을 불러온다.
7. AI가 큰 태스크를 쪼갠다.
8. 현재 실행 컨텍스트에 맞는 다음 행동을 만든다.
9. 작업 후보에 추가한다.

### 예시

입력:

"수집 시스템 정리해야 함"

AI 처리:

- 프로젝트: Personal Work OS
- 유형: 해야 할 일
- 분해:
  - 입력 유형 정리
  - 프로젝트 연결 규칙 정리
  - 맥락 질문 규칙 정리
- 현재 행동:
  - 수집 입력 예시 3개 적기

## 4. 프로젝트를 처음 등록하는 흐름

### 목적

AI가 태스크를 제대로 판단할 수 있도록 프로젝트의 최소 맥락을 만든다.

### 흐름

1. 사용자가 새 프로젝트를 만든다.
2. AI가 목표를 묻는다.
3. 사용자가 최종적으로 만들고 싶은 결과물을 말한다.
4. AI가 현재 어디까지 진행됐는지 묻는다.
5. 사용자가 지금까지 만든 것 또는 진행 상태를 말한다.
6. AI가 프로젝트 상태 카드를 만든다.
7. 사용자가 목표와 현재 상태를 확인한다.

### 최소 입력

- 최종 목표
- 현재까지 만든 것

### AI가 생성하는 것

- 상태 단계
- 현재 맥락 한 문장
- 다음 작은 결과물 후보
- 다음 10분 행동 후보

## 5. 작업을 시작하는 흐름

### 목적

작업 선택 부담을 줄이고 바로 실행으로 들어가게 한다.

### 흐름

1. 사용자가 오늘 작업 화면에서 추천 작업을 본다.
2. 추천 이유를 확인한다.
3. 필요하면 실행 컨텍스트 바를 바꾼다.
4. 추천 목록이 갱신된다.
5. 사용자가 작업을 시작한다.
6. 작업은 진행 중 상태가 된다.

### 추천 작업 카드

- 작업 제목
- 프로젝트
- 이유
- 시작 버튼
- 다른 추천 보기

## 6. 작업을 완료하는 흐름

### 목적

작업 맥락을 쌓되 기록 부담을 낮춘다.

### 흐름

1. 사용자가 완료 버튼을 누른다.
2. 시스템이 묻는다: "방금 한 작업을 한 줄로 남길래?"
3. 사용자는 텍스트/음성으로 남기거나 스킵한다.
4. 입력이 있으면 AI가 높은 신뢰도 로그로 정리한다.
5. 스킵하면 AI가 낮은 신뢰도 자동 최소 로그를 남긴다.
6. 프로젝트 상태 카드가 필요한 경우 업데이트된다.

## 7. 흐름이 깨졌을 때 복귀하는 흐름

### 목적

유튜브/멍함/회피 상태에서 다시 작업 가능한 상태로 돌아온다.

### 흐름

1. 사용자가 복귀 모드에 들어간다.
2. 시스템이 5분 유산소 복귀 행동을 제안한다.
3. 사용자가 버피 또는 대체 유산소를 한다.
4. 오늘 작업 화면으로 돌아온다.
5. 작업 강도는 `가볍게`로 설정된다.
6. AI가 가벼운 작업 3개를 추천한다.
7. 사용자가 하나를 시작한다.

## 8. 주간 계획 흐름

### 목적

모든 프로젝트를 같은 무게로 보지 않고, 이번 주에 전진/유지/보류/막힘을 정한다.

### 흐름

1. 사용자가 주간 판단 화면을 연다.
2. AI가 프로젝트별 최근 진척과 결과물을 요약한다.
3. AI가 전진/유지/보류/막힘을 제안한다.
4. 각 제안에는 근거가 붙는다.
5. 사용자가 제안을 수정하거나 확정한다.
6. 이번 주 전진 프로젝트 1~2개가 정해진다.
7. 각 전진 프로젝트의 다음 작은 결과물과 목표일을 정한다.

## 9. 목표일이 늦어지는 흐름

### 목적

늦어짐을 실패로 보지 않고 현실에 맞게 재조정한다.

### 흐름

1. AI가 목표일 지연 신호를 감지한다.
2. AI가 새 목표일을 제안한다.
3. AI가 변경 이유를 설명한다.
4. 사용자가 확정하거나 수정한다.
5. 목표일 변경 이유가 프로젝트 기록에 남는다.
6. 이후 유사한 계획 제안에 반영된다.

## 10. 데드라인 없는 프로젝트를 관리하는 흐름

### 목적

데드라인이 없어서 몇 주 동안 결과물 없이 흐르는 것을 막는다.

### 흐름

1. AI가 프로젝트의 마지막 결과물 날짜를 본다.
2. 몇 주간 결과물이 없으면 지연 위험을 표시한다.
3. AI가 작은 결과물 하나를 제안한다.
4. AI가 목표일 하나를 제안한다.
5. 사용자가 확정한다.
6. 오늘 작업 화면에 관련 10분 행동이 추천된다.

## 11. MVP에서 가장 중요한 시나리오

가장 먼저 검증해야 할 시나리오:

1. 사용자가 할 일을 입력한다.
2. AI가 프로젝트에 연결한다.
3. AI가 현재 환경에서 가능한 작업으로 바꾼다.
4. 오늘 작업 화면에 추천한다.
5. 사용자가 실행한다.
6. 완료 로그가 프로젝트 이해에 반영된다.

이 흐름이 자연스럽게 동작하면 MVP의 핵심 가치는 검증된다.
