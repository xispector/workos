# MVP Functional Spec

작성일: 2026-07-05

## 1. MVP 범위

MVP는 개인용 웹 앱이다. 목표는 사용자가 자신의 프로젝트와 해야 할 일을 입력하고, AI가 프로젝트 맥락을 바탕으로 지금 가능한 작업을 추천하며, 작업 완료 로그를 통해 프로젝트 이해를 갱신하는 것이다.

앱의 기본 언어는 한국어다. 화면 문구, 버튼, 안내 메시지, AI 응답, 샘플 데이터는 한국어로 제공한다. 내부 데이터 필드명이나 API 계약에서 필요한 기술 식별자는 영어를 사용할 수 있다.

## 2. 핵심 화면

MVP 화면은 5개로 시작한다.

1. 오늘 작업 화면
2. 빠른 수집 플로우
3. 프로젝트 대시보드
4. 프로젝트 상세 화면
5. 주간 판단 화면

계획 조정 기능은 프로젝트 상세와 주간 판단 안에 포함한다.

## 3. 오늘 작업 화면

### 목적

앱을 열었을 때 "지금 무엇을 해야 하는지"를 바로 보여준다.

### 구성

1. 상단 실행 컨텍스트 바
2. 추천 작업 3개
3. 빠른 수집 입력
4. 이번 주 전진 프로젝트 요약

### 상단 실행 컨텍스트 바

항상 보인다.

형식:

`집 · 노트북 · 조용함 · 보통`

필드:

- 장소: 집 / 외부 / 이동 중
- 도구: 노트북 가능 / 휴대폰만
- 환경: 조용함 / 산만함
- 작업 강도: 가볍게 / 보통 / 깊게

사용자가 직접 바꾼다. AI는 이 값을 기준으로 작업을 추천한다.

### 추천 작업 카드

기본 3개를 보여준다.

각 카드 필드:

- 작업 제목
- 연결 프로젝트
- 작업 강도
- 추천 이유 한 줄
- 시작 버튼
- 다른 추천 보기

추천 이유 예:

현재 `집 · 노트북 · 조용함 · 보통`이고, 이번 주 전진 프로젝트의 다음 작은 결과물과 연결되어 있어 추천함.

## 4. 빠른 수집 플로우

### 목적

사용자가 해야 할 일, 생각, 로그, 아이디어를 부담 없이 입력한다.

### 입력 방식

- 텍스트
- 음성

MVP에서는 파일, 이미지, 화면 녹화는 제외한다.

### 처리 흐름

1. 사용자가 자유 입력한다.
2. AI가 입력 유형을 추정한다.
3. AI가 프로젝트 연결을 시도한다.
4. 불확실하면 좁은 질문 하나를 한다.
5. 프로젝트 맥락을 불러온다.
6. 큰 태스크라면 3~5단계로 분해한다.
7. 현재 실행 컨텍스트에 맞는 다음 행동을 만든다.
8. 오늘 작업 후보에 추가한다.

### 입력 유형

초기 유형:

- 해야 할 일
- 프로젝트 로그
- 아이디어
- 공부/개념 메모
- 연락/네트워킹 메모
- 고민/결정 필요
- 일반 메모

사용자가 직접 고르지 않아도 된다. AI가 제안하고 필요할 때만 확인한다.

## 5. 프로젝트 대시보드

### 목적

AI에게 묻지 않아도 사용자가 전체 프로젝트 상태를 직접 스캔할 수 있게 한다.

### 구성

1. 상단 요약 스트립
2. 주의 필요 섹션
3. 프로젝트 리스트
4. 기본 필터

### 상단 요약 스트립

표시 항목:

- 전진 프로젝트 수
- 막힘 프로젝트 수
- 지연 위험 프로젝트 수
- 최근 7일 내 결과물 수

### 주의 필요 섹션

포함 조건:

- 막힘 상태
- 지연 위험
- 목표일 임박 + 최근 높은 신뢰도 로그 없음
- 2주 이상 결과물 없음
- 다음 10분 행동 없음

### 프로젝트 리스트

각 프로젝트 행에는 다음을 표시한다.

- 프로젝트 이름
- 상태 단계
- 이번 주 역할
- 현재 맥락 한 문장
- 다음 작은 결과물
- 다음 10분 행동
- 마지막 높은 신뢰도 로그 날짜
- 목표일 또는 지연 위험

### 기본 필터

- 전체
- 전진
- 유지
- 보류
- 막힘
- 지연 위험

## 6. 프로젝트 상세 화면

### 목적

프로젝트를 다시 열었을 때 목표, 현재 상태, 최근 진척, 다음 행동을 바로 이해하게 한다.

### 프로젝트 상태 카드

필드:

- 프로젝트 이름
- 최종 목표
- 상태 단계
- 현재 맥락 한 문장
- 결과물 목록
- 최근 진척 요약
- 다음 작은 결과물
- 다음 10분 행동
- 막힌 점
- 목표일
- 최근 로그

### 상태 단계

초기 상태:

- 대기
- 조사 중
- 설계 중
- 실행 중
- 막힘
- 완료

### 결과물 목록

프로젝트에서 실제로 남은 산출물을 보여준다.

예:

- 수집 시스템 요구사항 문서
- 첫 화면 와이어프레임 초안
- AI 질문 규칙 초안

AI가 결과물 추가를 제안할 수 있지만, 중요한 추가는 사용자가 확인한다.

### 최근 진척 요약

최근 높은 신뢰도 로그를 바탕으로 AI가 1~3문장으로 요약한다.

낮은 신뢰도 자동 로그는 "최근 작업 여부"에는 반영하지만, 진척 요약의 핵심 근거로는 약하게만 쓴다.

## 7. 주간 판단 화면

### 목적

이번 주에 무엇을 전진시키고, 무엇을 유지/보류/막힘으로 둘지 정한다.

### 프로젝트 역할

- 전진
- 유지
- 보류
- 막힘

### AI 제안 필드

각 프로젝트별로:

- 제안 역할
- 판단 근거
- 최근 진척 근거
- 지연 위험
- 다음 작은 결과물
- 목표일 제안 또는 조정 제안

### 사용자 조정

사용자는 AI 제안을 그대로 확정하거나 수정한다.

수정 가능한 항목:

- 역할
- 다음 작은 결과물
- 목표일
- 보류 사유

## 8. 작업 완료 로그

### 완료 후 기본 질문

"방금 한 작업을 한 줄로 남길래?"

사용자는 다음 중 하나를 선택한다.

- 텍스트 입력
- 음성 입력
- 스킵

### 로그 정리

사용자가 입력하면 AI가 다음으로 정리한다.

- 한 일
- 바뀐 상태
- 다음 행동
- 막힌 점

### 자동 최소 로그

사용자가 스킵하면 자동 최소 로그를 만든다.

형식:

`자동 기록` - `{작업 제목}` 작업을 완료로 표시함. 세부 로그 없음.

## 9. 로그 신뢰도

### 높은 신뢰도

조건:

- 사용자가 직접 텍스트 또는 음성으로 맥락을 남김
- AI가 정리함

사용처:

- 최근 진척 요약
- 프로젝트 상태 업데이트
- 다음 행동 갱신
- 목표일 조정 근거

### 낮은 신뢰도

조건:

- 사용자가 로그를 스킵함
- 작업 제목만 기반으로 자동 기록

사용처:

- 최근 작업 여부
- 히스토리

제한:

- 프로젝트 상태를 크게 바꾸지 않음
- 결과물 생성 근거로 쓰지 않음
- 목표일 조정의 강한 근거로 쓰지 않음

## 10. 계획 조정 기능

### 목표일 제안

프로젝트의 다음 작은 결과물에 대해 AI가 목표일 하나를 제안한다.

제안에는 다음이 포함된다.

- 목표일
- 근거
- 불확실성

### 목표일 재계산

조건:

- 목표일이 지났는데 결과물이 완료되지 않음
- 목표일 3일 전인데 최근 높은 신뢰도 로그가 없음
- 사용자가 늦어질 것 같다고 입력함
- 로그에 막힘/범위 증가/시간 부족이 나타남

출력:

- 기존 목표일
- 새 목표일
- 변경 이유
- 조정하지 않을 때의 위험

## 11. MVP 성공 플로우

MVP에서 반드시 동작해야 하는 플로우:

1. 사용자가 해야 할 일을 입력한다.
2. AI가 프로젝트에 연결한다.
3. AI가 필요한 경우 질문 하나로 맥락을 보강한다.
4. AI가 태스크를 현재 환경에서 가능한 작업으로 변환한다.
5. 오늘 작업 화면에 추천한다.
6. 사용자가 작업을 시작하고 완료한다.
7. 로그가 프로젝트에 쌓인다.
8. 프로젝트 상세 화면의 최근 진척과 다음 행동이 업데이트된다.
